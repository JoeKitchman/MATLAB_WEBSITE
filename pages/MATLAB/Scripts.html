MATLAB - Scripts & Functions

❮ Previous Next ❯

Although code can be directly input into the Command Window in MATLAB, this is not the best practice in some instances, as code can easily be lost and the window become cluttered. Scrips allow you to save sections of code to load them again later. This makes it easier to manage your code, and recall this section of code when needed, especially if it is required often.

Functions are even more powerful than that as they allow you to input values into a mathematical problem. This avoid the issues of having to manually change the value of a script when you need a new calculation

When a script or function has been saved, it can be recalled again by typing the name of the script within the Command Window or clicking the run button on the toolbar.

In the below examples, scripts and functions are demonstrated with the problem of calculating the area of a triangle.

Script

Scripts store commands exactly as you would type them at the command line in the command window. Adding this section of code to a script will calculate the area of a triangle with a base length of 5 and a height of 3:

b = 5;
h = 3;
a = 0.5*(b*h)

After saving the file as "triarea", you can call the script from the command line:

triarea
a = 
    7.5000

The main downside to a script is you can not change the values without manually editing it. This is where functions become more valuable.

Function

Functions are much more flexible, allowing you to input values dynamically. Taking the script used above, you can turn it into a function very easily:

function a = triarea(b,h)
a = 0.5*(b*h);
end

Calling the function in the command window as you would use any other function, will run the function with the inserted values:

a = triarea(5,3)

This then produces the following result:

a = 
    7.5000
